# Aurora V4 demo plan

Record with a clean test account and dummy data. Do not show API keys, ChatGPT
account details, private memory files, personal mail, notifications, tailnet
addresses, pairing codes, signing identities, or local logs.

## 60–90 second application cut

1. **0:00–0:08 — Problem.** “Most voice agents lose continuity or become
   unbounded tool wrappers. Aurora keeps conversation, memory, and action on
   explicit boundaries.”
2. **0:08–0:25 — Voice.** Wake Aurora and have one natural spoken exchange.
3. **0:25–0:45 — Delegation.** Give one safe, visually legible task. Show that
   Aurora acknowledges it once, stays responsive, and delegates through Codex.
4. **0:45–0:58 — Verified result.** Show the completed effect, then Aurora's
   short outcome report. Do not expose Codex reasoning or credentials.
5. **0:58–1:10 — Continuity.** Rest and wake, then demonstrate one remembered
   commitment or correctly grounded relationship detail using dummy content.
6. **1:10–1:20 — Evidence.** End on the architecture diagram and a successful
   verifier result. State one limitation: this release has no notarized public
   installer and excludes the private iPhone companion prototype.

## Two-to-three-minute technical cut

Use the same real interaction, then briefly explain:

- why Realtime owns foreground conversation;
- how `delegate_task` becomes action-scoped authorization;
- why observed content cannot widen the task;
- how executor state and effect evidence gate completion claims;
- how bounded continuity avoids dumping every memory into the prompt;
- which deterministic tests run in CI and which live checks remain local.

## Release checklist

- Record the exact locally verified build, not an older download archive.
- Add captions and trim waiting time; do not replace real behavior with slides.
- Upload both cuts and place the 60–90 second link near the top of `README.md`.
- Add the short demo and GitHub links to the portfolio's Aurora case study.
- Verify every link in a signed-out browser before using it in applications.
